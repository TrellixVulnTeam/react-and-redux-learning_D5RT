Author: John Kirkham <kirkhamj@janelia.hhmi.org>
Date:   Wed Mar 14 17:01:28 2018 -0400

    Install locally built anaconda-client
    
    After building `anaconda-client`, install our newly built copy of
    `anaconda-client`. Make sure to do this right after building and before
    uploading. That way we will attempt to upload our new package of
    `anaconda-client` using our new build of `anaconda-client`. IOW this is
    a great test of whether `anaconda-client` is defective. If it turns out
    `anaconda-client` cannot upload itself with our upload script, it will
    cause CIs to error before the upload to occur. Thus avoiding publishing
    copies of `anaconda-client` that would break other feedstocks. We even
    get some of these testing benefits before merging (though obviously not
    the actual uploading part).
